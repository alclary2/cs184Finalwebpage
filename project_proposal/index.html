Final Project Proposal



Title: Extra Trippy NonEuclidean World

Team Members: Anna Clary, Ava Patel, Lucy Lou, Iman Kahssay

Summary

Using existing code from a github user named “HackerPoet”, we are able to render a non-euclidean world where natural spatial discernment is nonexistent. Our goal is to implement features that make the scenes look more realistic using textural procedures, and to procedurally generate the rooms in the house.



Problem Description

Although the setup of the “world” is already given, the environment is not rendered to look realistic. The textures and colors in the scene are nonexistent. So, our group’s goal is to implement textural procedures we have learnt in lecture and project one to make the environment look more realistic to the user. Along with this, we plan to make the world less deterministic; we will render each floor plan randomly as the user steps through them in the game. Not only do we plan to make this world look more realistic but also more perplexing (and interesting). This plan is not only important in rendering realistic real-time graphics, but it is very intriguing to both the user/player and designer.



Goals and Deliverables

What you plan to deliver?

  • Different textures/shadings
    • - Each room has a mesh (such as a bunny, teapot, etc.) shown inside it. We can implement shadings into the rooms and textures onto the meshes that appear in the rooms.
    - Implement mipmap levels when rendering textures.
  • Procedural generation of rooms

What you hope to deliver?

  • Possibly create the penrose staircase
  • Procedural generation combining multiple non-euclidean aspects
  • Lighting/raytracing
